```
✅ comment table: id, tweet_id, user_id, text, created_at, parent_id
✅ tweet_likes (M2M): tweet_id, user_id
✅ notification table: id, user_id, comment_id, notification_type, is_read, created_at
✅ All indexes applied
```

Production Database (Supabase - Currently):

```
❌ tweet table: id, user_id, text, photo (OLD!), created_at, updated_at
   MISSING: view_count ← CAN'T WRITE TO FIELD | CAN'T READ VIEW STATS

   MISSING: parent_id ← CAN'T CREATE NESTED REPLIES
```
